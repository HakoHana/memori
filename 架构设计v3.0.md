> 版本：v3.0 — 2026-06-09

---

## 一、概述

Memory AstrBot 是一个日记式长期记忆插件，为 AstrBot 提供以下能力：

- **日记式记忆**：LLM 以第一人称写日记，记录重要对话时刻
- **原子事实**：从日记提取结构化事实，支持全文检索
- **知识图谱**：自动构建实体关联图，可视化浏览
- **用户画像**：长期沉淀的用户特征画像
- **WebUI Dashboard**：记忆管理、图谱可视化、日记编辑

## 四、三层缓存架构（实际实现）

```
     ┌─────────────────────┐
     │  热缓存 (HotCache)    │ deque 内存，每用户 ≤20 条
     │  命中即返回，不查 DB  │ Retriever.get_recent_context() 优先读
     └─────────┬───────────┘
               │ 未命中
     ┌─────────▼───────────┐
     │  温层 (WarmProcessor) │ asyncio.Queue → Judge → Capture → L3
     │  异步后台，不阻塞回复  │ 触发条件：距上次整理 > 间隔阈值
     └─────────┬───────────┘
               │ 整理完成
     ┌─────────▼───────────┐
     │  冷层 (Archiver)      │ >90天 且 importance<0.1 → Markdown 归档
     │  定时扫描             │ 导出 memory_archive/{uid}/YYYY/MM/
     └─────────────────────┘
```

## 五、核心表结构（关键字段）

### diary_entries（L1 日记 — 唯一真相源）

```sql
diary_entries(
    id, uid, user_id, date, timestamp, content,          -- 基础
    importance, mood, topics,                              -- frontmatter 字段
    fact_extracted, fact_retry_count,                      -- L2 状态
    archived, correction                                   -- 归档/更正标记
)
```

### atomic_facts（L2 原子事实 — 全局去重）

```sql
atomic_facts(
    id, uid, user_id,
    content UNIQUE,                                        -- 全局去重键
    atom_type, importance, confidence,
    source_count, fact_version
)
```

### diary_fact_links（L1↔L2 多对多）

```sql
diary_fact_links(
    diary_id, fact_id,
    snippet, importance_in_diary,
    PRIMARY KEY (diary_id, fact_id)
)
```

### user_persona（L3 画像）

```sql
user_persona(
    uid PRIMARY KEY,
    summary, full_markdown,                                -- ≤200 tokens 摘要 / 完整版
    known_ids, primary_name, identity_confidence,
    tier,                                                  -- core/active/occasional/new
    version, last_full_update, last_incremental_update
)
```

### graph_nodes / graph_edges（L4 知识图谱）

```sql
graph_nodes(id, node_key UNIQUE, node_type, value, canonical_value, metadata)
-- node_type: entity / user / diary / topic / emotion / date

graph_edges(id, edge_key UNIQUE, semantic_key,
            source_node_id, target_node_id,
            relation_type, source_memory_id,
            weight, confidence, status)
-- relation_type: mentions / co_occur / relates_to / same_user
```

### entity_cooccur（共现预计算 — O(1) 查询）

```sql
entity_cooccur(
    entity_a_id, entity_b_id,
    count, last_updated,
    PRIMARY KEY (entity_a_id, entity_b_id)
)
```

### canonical_users / user_identities（身份体系）

```sql
canonical_users(uid PRIMARY KEY, primary_name, identity_confidence)
user_identities(platform_id PRIMARY KEY, uid, platform, display_name, verified)
```

## 六、写入流水线（WarmProcessor）

```
WarmProcessor.worker()          # asyncio.Queue 消费者
  │
  ├─ 1. judge_provider.should_capture(msg) → 是否值得记？
  │       └─ false → 跳过
  │
  ├─ 2. capturer.capture(msg)
  │       ├─ _write_diary()           → diary_entries 写入 L1
  │       ├─ _extract_atoms()         → LLM 提取原子事实
  │       │     ├─ 去重（content UNIQUE）
  │       │     ├─ diary_fact_links 关联
  │       │     └─ 重要度排序限 top 5
  │       └─ on_atoms_created()       → fire-and-forget 图谱索引
  │
  ├─ 3. graph_engine.index_diary()
  │       ├─ 解析 [[实体]]
  │       ├─ 创建 diary 节点
  │       ├─ 创建/更新 entity/user 节点
  │       ├─ 创建 mentions 边
  │       └─ 更新 entity_cooccur
  │
  └─ 4. persona_engine.incremental_update()
          └─ 每 N 篇日记触发 → LLM diff → 更新画像
```

## 七、读取流水线（MemoryCore + Retriever）

```
用户消息到达
  │
  ├─→ hot_cache.get_recent_context(uid)
  │     命中？ → 直接返回
  │
  ├─→ retriever.hybrid_search(query, uid)
  │     ├─ 意图分类：
  │     │    "关系"/"关联"/"共同" → 走图查询
  │     │    其他 → 走表查询
  │     │
  │     ├─ 表查询：FTS5 + LIKE（中文正则分词）
  │     ├─ 图查询：cooccur (O(1)) / 邻居 (索引)
  │     └─ RRF 多路融合 + 综合评分排序
  │
  └─→ memory_injector.inject(context, search_results)
         └─ 拼入 req.system_prompt → LLM 回复时可用
```

## 八、用户分层（Tier）

| 等级 | 条件 | 预过滤 | LLM 判断 | 画像更新 |
|------|------|--------|---------|---------|
| core | 近 7 天有消息 | 跳过 | 每次 | 每 N 篇 |
| active | 近 30 天有消息 | 跳过 | 每次 | 每 N 篇 |
| occasional | 近 90 天有消息 | 走过滤 | 仅关键词 | 不更新 |
| new | >90 天或新用户 | 走过滤 | 仅关键词 | 不更新 |

预过滤规则（仅 occasional/new）：
- 长度 < 3 → 丢弃
- 重复率 > 60% → 丢弃
- 纯 emoji → 丢弃
- 与本人上条 30s 内相似 → 丢弃

## 十、配置项（核心）

```yaml
memory:
  hot_cache_size: 20           # 每用户热缓存消息数
  consolidate_interval: 120    # 整理间隔（秒）
  judge_provider: ""           # 判读模型（可与主模型分离）
  capture:
    max_atoms_per_capture: 5   # 每条日记最多提取原子数
    pre_filter_enabled: false  # 预过滤开关
    tier_bypass_prefilter: ["core", "active"]
  persona:
    update_trigger_diary_count: 5
    ttl_seconds: 60
  archive:
    enabled: true
    cold_days: 90
    importance_threshold: 0.1
```
